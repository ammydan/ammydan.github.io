搭建博客

Author: CreatorDP Date: September 24, 2020  12:50:15

无业在校码农搭建一个长期博客，肯定要先看看有没有免费的路子，实在不行就只能自己买一个便宜的服务器搭建了。好在有这么一个路子，github pages+hexo/其他静态网站生成器——>0元即可拥有一个静态站点。

简单一套流程

下载以及配置Node.js

Node.js在官方的定义是js运行环境还顺带包含了一些命令行指令。（这样我们就可以脱离浏览器来运行JS了，基本上和Java，python运行环境一致，npm类似于python的pip）

我们来下载Node.js

配置npm

因为npm默认配置的是国外仓库的源速度太慢，所以我们需要配置好国内镜像源。

  • 更换为阿里的镜像源
npm install -g cnpm --registry=https://registry.npm.taobao.org
利用cnpm下载安装hexo
cnpm install -g hexo-cli

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

为了验证我们是否安装成功，可以使用以下命令行

hexo -v

使用hexo

生成一个博客项目

  1. 创建一个文件夹

    这个用来保存你的博客项目的，然后你需要把你的命令框的所在位置移到对应文件夹下。（注意你必须先清空文件夹）

  2. 初始化一个项目

    确认你已经在该文件下，并且文件为空。

hexo init

Github pages

创建一个gitbub pages 项目
  1. 安装git部署插件
cnpm install --save hexo-deployer-git

  1. 配置文件设置
    _config.yaml

    # Deployment
## Docs: https://hexo.io/docs/one-command-deployment
deploy:
 type: git
 repo: https://github.com/ammydan/ammydan.github.io.git
 branch: master

这个自动部署插件会复制，public文件下的所有内容来部署，如果你想要自己添加一些网页内容，你可以放在public文件下，那么你的插件会帮你一同部署。

hexo d

更换主题

寻找合适的主题
  • 我们可以去Hexo官网看看自己喜欢那个主题，然后找到对应的github地址。
  • 也可以直接google搜寻
克隆主题仓库
  • 在主题的github首页赋值其地址
  • 在本地clone对应的主题到themes文件夹下
  • 修改_config.yaml中的theme为对应主题的文件夹名
  • 命令行重启
    hexo clean
hexo g
hexo s

HEXO

文件结构

scaffolads

markdown模板，里面有三种草稿(draft)、网页(page)、博客(post)。我们可以自行修改文件的格式，那么每次生成的新文件都会自动添加的这些信息。当然我们也可以创建自己的模板，然后根据自己模板选择不同的样式layout。

source

这里放置我们所有的markdown文件，_post放置我们博客markdown文件，当然也会有page。

themes

这是放置主题样式的文件夹。

_config.yml

这里设置一些hexo的参数，这里枚举一下常用的参数：

  • theme: xxx 设置主题样式
  • deploy: …. 设置部署方式
.gitignore

这个应该很熟悉，文件写的是git提交时会忽略的文件和文件夹。

package.json

这是hexo的依赖库记录

node_modules

这个是hexo的依赖库

开发命令行指令

1. 进行本地测试hexo，在对应root文件夹下。
hexo server
2.创建一个post、草稿、page
hexo new <name>
hexo new draft <name>
hexo new page <name>

//将草稿变成一个博客
hexo publish <name>

tags 和 categories

tags
tags: [tag1, tag2, tag3]

HEXO会根据tags创建对应的文件夹。

categories
categories: 
- [cat1, cat1.1]
- [cat2]

HEXO也会根据categories创建文件夹。

tag plugin

添加代码块
{% codeblock lang: javascript %}
	alert("Hello world!");
	var myVar = "Hello World";
{% endcodeblock %}
添加Youtube视频
{% youtube hY7m5jjJ9mM %}

asset

我们的markdown文件往往需要引用许多图片，所以我们在部署我们的静态网页的时候也要将图片部署上去。为了在创建一个markdown文件时也创建静态资源文件夹，我们需要在_config.yml中设置

post_asset_folder: true

创建自己的主题样式

主题样式创建

创建主题文件夹
  1. 在themes中创建自己样式文件夹
  2. 在themes中我们需要创建以下文件
    • languages
    • layout
    • script
    • source
    • config.yml

layout

Template Page Fallback

主布局

layout.ejs是主模板，body会根据网页到底是博客、首页还是网页以及归档和分类等来按需传入。

<!DOCTYPE html>
<html>
  <body><%- body %></body>
</html>
局部模板

局部模板让您在不同模板之间共享相同的组件，例如页首（Header）、页脚（Footer）或侧边栏（Sidebar）等，可利用局部模板功能分割为个别文件，让维护更加便利。举例来说：

partial/header.ejs
<h1 id="logo"><%= config.title %></h1>
index.ejs
<%- partial('partial/header') %>
<div id="content">Home page</div>
参数

您可以在局部模板中指定局部变量并使用。

partial/header.ejs
<h1 id="logo"><%= title %></h1>
index.ejs
<%- partial('partial/header', {title: 'Hello World'}) %>
<div id="content">Home page</div>

生成：

<h1 id="logo">Hello World</h1>
<div id="content">Home page</div>

变量

变量可以来自于整个web site也可以来自于该模板页面。

全局变量
变量 描述 类型
site 网站变量 object; 见 网站变量
page 针对该页面的内容以及 front-matter 中自定义的变量。 object; 见 页面变量
config 网站配置 object (站点的配置文件)
theme 主题配置。继承自网站配置。 object (主题配置文件)
path 当前页面的路径（不含根路径） string
url 当前页面的完整网址 string
env 环境变量 ???
网站变量
变量 描述 类型
site.posts 所有文章 array of post objects
site.pages 所有分页 array of page objects
site.categories 所有分类 object，包含了站点全部的分类
site.tags 所有标签 array，包含了站点全部的标签
页面变量

页面（page）

变量 描述 类型
page.title 页面标题 string
page.date 页面建立日期 Moment.js 对象
page.updated 页面更新日期 Moment.js 对象
page.comments 留言是否开启 boolean
page.layout 布局名称 string
page.content 页面的完整内容 string
page.excerpt 页面摘要 string
page.more 除了页面摘要的其余内容 string

page.full_source 页面的完整原始路径 string
page.path 页面网址（不含根路径）。我们通常在主题中使用 url_for(page.path)。 string
page.permalink 页面的完整网址 string

page.raw 文章的原始内容 ???

page.link 文章的外部链接（用于链接文章） string

meta标签

meta 元素可提供有关页面的元信息（meta-information），比如针对搜索引擎和更新频度的描述和关键词。meta 标签位于文档的头部，不包含任何内容。meta 标签的属性定义了与文档相关联的名称/值对。

属性 值 描述
http-equiv content-typeexpiresrefreshset-cookie 把 content 属性关联到 HTTP 头部。
name authordescriptionkeywordsgeneratorrevisedothers 把 content 属性关联到一个名称。
scheme some_text 定义用于翻译 content 属性值的格式
meta Http-equiv

http-equiv顾名思义，相当于http的文件头作用，它可以向浏览器传回一些有用的信息，以帮助正确和精确地显示网页内容，与之对应的属性值为content，content中的内容其实就是各个参数的变量值。

<meta http-equiv="charset" content="iso-8859-1">
<meta http-equiv="expires" content="31 Dec 2008">
viewport
<meta name="viewport" content="width=device-width, initial-scale=1, shrink-to-fit=no">
