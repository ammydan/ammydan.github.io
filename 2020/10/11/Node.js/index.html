设置安装目录

我们默认的全局安装目录是在C盘下面的，如果我们想要自定义安装目录需要用以下命令

npm config set prefix "<目录>"

npm config set cache "目录"

#检查是否已经更行
npm config get prefix
npm config get cache

这个是设置的全局安装路径，如果你要安装全局路径则需要在npm安装命令添加-g参数

npm install -g <安装包>

为了方便查找全局安装包的路径，我们需要在系统环境中配置NODE_PATH（路径iu是我们设置的prefix的node_modules包下，这个包尚未创建，但是安装包的时候就会自动创建）。
注意这个环境需要重启电脑后才能生效。

如何从Node.js程序退出

最简单粗暴的方式就是ctrl-C，但是并不推荐这么做。我们可以在代码中添加process.exit()来让程序自动退出。

process.exit(1);

如果你想要通过命令行发送signal给程序让其停止，可以加上以下代码

process.on('SIGTERM', () => {

    console.log('Process terminated')
  })
})

Node.js 单线程模型（事件循环模型）

这里可以参考我在操作系统分类中的事件多线程模型（那篇文章更加通俗易懂，并且提供了简单的代码加强理解），使用这种模型的一个最大优点就是让程序员不用考虑复杂环境下的并发问题，而且无需担心使用锁之后的死锁问题。通常浏览器中，每个浏览器选项卡中都有一个事件循环。

JS基本事件概念

调用堆栈

调用堆栈是一个LIFO队列。事件循环不断地检查调用堆栈，以查看是否需要运行任何函数。当执行时，它会将找到地所有函数添加到调用堆栈中，并按顺序执行每个函数。

消息队列

当调用setTime()时，浏览器或者Node.js会启动定时器。当定时器到期时，则回调函数会被放入“消息队列”中。在消息队列中，用户触发的事件也会在此排队，然后代码才有机会对其做出反应。

事件循环会赋予调用堆栈优先级，它首先处理在调用堆栈中找到的所有东西，一旦其中没有任何东西，便开始处理消息队列中的东西。

ES6作业队列

这种方式会尽快地执行一步函数的结果，而不是放在调用堆栈的末尾。在当前函数结束之前 resolve 的 Promise 会在当前函数之后被立即执行。也就是这次调用堆栈结束之前会执行Promise。

const bar = () => console.log('bar')

const baz = () => console.log('baz')

const foo = () => {
  console.log('foo')
  setTimeout(bar, 0)
  new Promise((resolve, reject) =>
    resolve('应该在 baz 之后、bar 之前')
  ).then(resolve => console.log(resolve))
  baz()
}

foo()


//foo
//baz
//应该在 baz 之后、bar 之前
//bar

事件模型

process.nextTick()

一次循环就称为一次tick，当一个函数传给process.nextTick()时，则指示引擎在当前操作结束（在下一个事件循环tick开始之前）时调用此函数。

process.nextTick(() => {
  //做些事情
})

调用setTimeout(()=>{},0)会在下一个tick结束时执行该函数，比使用nextTick()晚得多。

setImmediate()

当要尽快执行某些代码时我们可以使用以下代码

setImmediate(()=>{
    //运行一些东西
})

这个和setTimeout(()=>{},0)基本差不多。

setTimeout
const myFunction = () => {
  // 做些事情

  setTimeout(myFunction, 1000)
}

setTimeout(myFunction, 1000)

Promise

Promise如何运作

当 promise 被调用后，它会以处理中状态开始。 这意味着调用的函数会继续执行，而 promise 仍处于处理中直到解决为止，从而为调用的函数提供所请求的任何数据。

被创建的 promise 最终会以被解决状态或被拒绝状态结束，并在完成时调用相应的回调函数（传给 then 和 catch）。

以下WEB API使用了promise

  • Battery API
  • Fetch API
  • Service Worker
创建promise
let done = true

const isItDoneYet = new Promise((resolve, reject) => {
  if (done) {
    const workDone = '这是创建的东西'
    resolve(workDone)
  } else {
    const why = '仍然在处理其他事情'
    reject(why)
  }
})

function asyncFunction() {

    return new Promise(function (resolve, reject) {
        setTimeout(function () {
            resolve('Async Hello world');
        }, 16);
    });
}

asyncFunction().then(function (value) {
    console.log(value);    // => 'Async Hello world'
}).catch(function (error) {
    console.log(error);
});

如你所见，promise 检查了 done 全局常量，如果为真，则 promise 进入被解决状态（因为调用了 resolve 回调）；否则，则执行 reject 回调（将 promise 置于被拒绝状态）。 如果在执行路径中从未调用过这些函数之一，则 promise 会保持处理中状态。

const fs = require('fs')

const getFile = (fileName) => {
  return new Promise((resolve, reject) => {
    fs.readFile(fileName, (err, data) => {
      if (err) {
        reject (err)  // 调用 `reject` 会导致 promise 失败，无论是否传入错误作为参数，
        return        // 且不再进行下去。
      }
      resolve(data)
    })
  })
}

getFile('/etc/passwd')
.then(data => console.log(data))
.catch(err => console.error(err))
Promise的状态

  • has-resolution- Fullfilled

    resolve，此时会调用onFullfilled

  • has-rejection- Rejected

    reject，此时会调用onRejected

  • unresolved- Pending

    既不是resolve也不是reject的状态。也就是promise对象刚被创建后的初始化状态等。

从Pending转换为Fullfilled或者Rejected之后，这个promise对象不会再发生任何变化。

使用promise
  1. new Promise(fn)返回一个promise对象
  2. 在fn 中指定异步等处理
    • 处理结果正常的话，调用resolve(处理结果值)
    • 处理结果错误的话，调用reject(Error对象)

Promise实战

比如 Promise.resolve(42); 可以认为是以下代码的语法糖。

new Promise(function(resolve){
    resolve(42);
});

方法 Promise.resolve(value); 的返回值也是一个promise对象，所以我们可以像下面那样接着对其返回值进行 .then 调用。

Promise.resolve(42).then(function(value){
    console.log(value);
});

比如 Promise.reject(new Error("出错了")) 就是下面代码的语法糖形式。

new Promise(function(resolve,reject){
    reject(new Error("出错了"));
});

这段代码的功能是调用该promise对象通过then指定的 onRejected 函数，并将错误（Error）对象传递给这个 onRejected 函数。

Promise.reject(new Error("BOOM!")).catch(function(error){
    console.error(error);
});
Promise chain
function taskA() {
    console.log("Task A");
}
function taskB() {
    console.log("Task B");
}
function onRejected(error) {
    console.log("Catch Error: A or B", error);
}
function finalTask() {
    console.log("Final Task");
}

var promise = Promise.resolve();
promise
    .then(taskA)
    .then(taskB)
    .catch(onRejected)
    .then(finalTask);

如何传递参数

function doubleUp(value) {
    return value * 2;
}
function increment(value) {
    return value + 1;
}
function output(value) {
    console.log(value);// => (1 + 1) * 2
}

var promise = Promise.resolve(1);
promise
    .then(increment)
    .then(doubleUp)
    .then(output)
    .catch(function(error){
        // promise chain中出现异常的时候会被调用
        console.error(error);
    });

return的值会由 Promise.resolve(return的返回值); 进行相应的包装处理，因此不管回调函数中会返回一个什么样的值，最终 then 的结果都是返回一个新创建的promise对象。也就是说， Promise#then 不仅仅是注册一个回调函数那么简单，它还会将回调函数的返回值进行变换，创建并返回一个promise对象。

var promise = Promise.reject(new Error("message"));
promise.then(undefined, function (error) {
    console.error(error);
});
Promise与数组

如果只有一个promise对象的话我们可以像前面介绍的那样编写代码就可以了，如果要在多个promise对象都变为FulFilled状态的时候才要进行某种处理话该如何操作呢？

Promise.all 接收一个 promise对象的数组作为参数，当这个数组里的所有promise对象全部变为resolve或reject状态的时候，它才会去调用 .then 方法。与之相对的是 Promise.race 只要有一个promise对象进入 FulFilled 或者 Rejected 状态的话，就会继续进行后面的处理。 Promise.race在第一个promise对象变为Fulfilled之后，并不会取消其他promise对象的执行。

var p1 = Promise.resolve(1),
    p2 = Promise.resolve(2),
    p3 = Promise.resolve(3);
Promise.all([p1, p2, p3]).then(function (results) {
    console.log(results);  // [1, 2, 3]
});
var p1 = Promise.resolve(1),
    p2 = Promise.resolve(2),
    p3 = Promise.resolve(3);
Promise.race([p1, p2, p3]).then(function (value) {
    console.log(value);  // 1
});
Promise.all([read_csv_file("devices.csv"), read_csv_file("bugs.csv")])
  .then(function(results) {
    var first = results[0];  // contents of the first csv file
    var second = results[1]; // contents of the second csv file
  });

Node.js 服务器

一个简单例子
const http = require('http');

const hostname = '127.0.0.1';
const port = 3000;

const server = http.createServer((req, res) => {
  res.statusCode = 200;
  res.setHeader('Content-Type', 'text/plain');
  res.end('Hello World');
});

server.listen(port, hostname, () => {
  console.log(`Server running at http://${hostname}:${port}/`);
});

HTTP模块创建一个HTTP服务器，该服务器在3000端口监听，当服务监听就绪时会调用回调函数。
