分布式多层次应用结构

应用软件逻辑根据功能分成不同的组件(components)，组成javaEE应用的组件被安装再不同的机器上（这取决于components处于javaEE的哪个层次）。

  • 客户层组件运行在客户电脑上
  • web层组件运行在java EE server上
  • 业务层组件运行在java EE server上
  • 数据层软件运行在EIS server上

Java EE Components

java EE组件是一个独立功能软件单位，它可以被整合到java EE应用中，也可以其他组件进行通信。Java EE标准定义了Java EE组件：

  • 客户端和applets运行在客户机器上
  • Java Servlet, JavaServer Faces, and JavaServer Pages (JSP) 技术组件是web组件运行在服务器上
  • EJB组件是业务逻辑组件运行在服务器上

Java EE 客户端

web客户端

一个web客户端包含两个部分:

  • 动态页面(HTML, XML)，是通过web组件生成的。
  • web浏览器，渲染页面。
应用客户端

应用客户端提供了大量GUI，并且使用了客户端的资源进行运算，减轻服务端的工作量。

JavaBeans 组件结构

JavaBeans组件是用来管理数据流的：

  • Java EE客户端组件与Java EE服务器之间
  • 服务器组件和数据库之间

JavaBean有许多属性，并且使用get和set方法进行获取。

JavaEE 服务端交流

WEB组件

web组件要么是servlet或者是由JSF或者JSP技术生成的web页面。Servlet是java动态处理request和生成responses的类；JSP是基于文本文件的，作为一个Servlet来运行，但是提供了一个更便捷的方式写静态页面代码。JSF是基于Servlet和JSP为web应用程序提供一个用户接口组件框架。

业务逻辑组件

业务代码是用来解决特殊的业务需求的，业务代码是被运行在web层或者业务层的enterprise组件处理的。enterprise beans可以从数据库获取数据，预处理数据并将数据返回给客户端。

JavaEE 容器

瘦客户端应用实现是非常复杂的，涉及到很多复杂的代码，比如：处理事务、状态管理、多线程、资源池以及其他一些复杂的底层细节。Java EE Server通过容器形式为组件提供了底层服务，这样开发人员无需再自己实现底层的服务。

容器功能

容器是组件和底层服务功能的一个接口，组件在运行之间必须整合到java EE module中并且部署到容器。

整合的过程涉及到每个组件的容器设置，这个设置会改变java EE的支持功能，比如：安全、事务管理、JNDI查询接口、远程连接等。

  • The Java EE security model lets you configure a web component or enterprise bean so that system resources are accessed only by authorized users.
  • The Java EE transaction model lets you specify relationships among methods that make up a single transaction so that all methods in one transaction are treated as a single unit.
  • JNDI lookup services provide a unified interface to multiple naming and directory services in the enterprise so that application components can access these services.(enables components to locate other components and resources.)
  • The Java EE remote connectivity model manages low-level communications between clients and enterprise beans. After an enterprise bean is created, a client invokes methods on it as if it were in the same virtual machine.

容器也涉及了一起其他的非配置服务，比如enterprise组件和servlet生命周期，数据库连接池，数据持久化和Java EE API访问。

容器类型

  • Java EE server: The runtime portion of a Java EE product. A Java EE server provides EJB and web containers.
  • EJB container: Manages the execution of enterprise beans for Java EE applications. Enterprise beans and their container run on the Java EE server.
  • Web container: Manages the execution of web pages, servlets, and some EJB components for Java EE applications. Web components and their container run on the Java EE server.
  • Application client container: Manages the execution of application client components. Application clients and their container run on the client.
  • Applet container: Manages the execution of applets. Consists of a web browser and a Java Plug-in running on the client together.
